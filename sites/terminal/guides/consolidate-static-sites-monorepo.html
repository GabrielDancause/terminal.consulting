How to Consolidate Multiple Static Sites into One Monorepo

2026-03-04

If you manage multiple static websites — each in its own GitHub repo, each deployed separately — you can consolidate them into a single monorepo and deploy each site independently with Cloudflare Pages.

This guide covers the repo restructuring. You end up with one repo, one place to manage all your sites, and the option to make it private.

The Problem

You have several static sites, each in its own public GitHub repo, each deployed via GitHub Pages:

# Separate repos, separate deploys
your-agency/          → yourdomain.com
your-agency/client-a  → client-a.com
your-agency/client-b  → client-b.com
your-agency/client-c  → client-c.com

This means public source code for every client, scattered repos, and no centralized management.

The Target Structure

your-repo/
├── sites/
│   ├── main/        ← yourdomain.com
│   │   ├── index.html
│   │   └── styles.css
│   ├── client-a/    ← client-a.com
│   │   ├── index.html
│   │   └── ...
│   ├── client-b/    ← client-b.com
│   │   └── ...
│   └── client-c/    ← client-c.com
│       └── ...
└── ...

One repo. Each site in its own directory. Each deployed to its own domain via Cloudflare Pages.

Step 1: Move Your Main Site

Move your existing site files into a subdirectory:

$ mkdir -p sites/main
$ mv index.html styles.css scripts.js sites/main/
$ rm CNAME  # Cloudflare handles the domain now

Step 2: Import Client Repos

For each client site, clone the repo and copy the files in (without git history):

$ git clone --depth 1 https://github.com/you/client-a.git /tmp/client-a
$ rsync -a --exclude='.git' --exclude='CNAME' /tmp/client-a/ sites/client-a/

Repeat for each site. The --depth 1 flag keeps the clone fast since you only need the latest files.

Step 3: Commit

$ git add sites/
$ git rm index.html styles.css scripts.js CNAME
$ git commit -m "Restructure into monorepo with all sites"

Step 4: Deploy with Cloudflare Pages

In the Cloudflare dashboard, create a Pages project for each site:

  1. Connect your GitHub repo
  2. Set the build output directory to the site's folder (e.g. sites/client-a)
  3. Add your custom domain
  4. Repeat for each site

Each Cloudflare Pages project watches the same repo but serves a different directory to a different domain.

Step 5: Clean Up

Once all sites are live on Cloudflare:

# Make the monorepo private
$ gh repo edit you/your-repo --visibility private

# Archive the old repos (keeps them as backup)
$ gh repo archive you/client-a
$ gh repo archive you/client-b
$ gh repo archive you/client-c

Things to Watch

  • DNS propagation — domain changes can take minutes to 48 hours
  • Git history — client repos' history won't carry over with this method. Use git subtree if you need it
  • Cloudflare free tier — 500 builds/month, 20,000 files per project, unlimited bandwidth
  • Repo size — watch for large assets. A monorepo with images across multiple sites can grow fast
