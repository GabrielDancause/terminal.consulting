How to Point a Custom Domain to Cloudflare Pages

2026-03-04

You deployed a site to Cloudflare Pages. It's live at your-project.pages.dev. Now you want it on your actual domain. The steps depend on where your DNS is managed.

Option A: Domain Already on Cloudflare DNS

This is the easy path. If Cloudflare is already your nameserver:

  1. Go to your Pages project → Custom domains → Set up a custom domain
  2. Enter your domain (e.g. yourdomain.com)
  3. Cloudflare adds the DNS record automatically
  4. SSL certificate is provisioned within minutes

That's it. Cloudflare handles the CNAME record and the certificate.

Option B: Domain on an External Registrar

If your DNS is managed elsewhere (Namecheap, GoDaddy, Google Domains, etc.), you have two choices:

Choice 1: Move your nameservers to Cloudflare (recommended)

  1. Add your domain to Cloudflare (free plan)
  2. Cloudflare scans your existing DNS records
  3. Update your registrar's nameservers to the ones Cloudflare gives you
  4. Wait for propagation (minutes to 48 hours)
  5. Then follow Option A above

Choice 2: Add a CNAME record at your current registrar

  1. At your registrar's DNS settings, add a CNAME record:
Type Name Target
CNAME @ or yourdomain.com your-project.pages.dev

Note: some registrars don't allow CNAME on the root domain (@). In that case, use the www subdomain and set up a redirect from the root, or move your nameservers to Cloudflare.

Setting Up www + Root Domain

Most sites should work on both yourdomain.com and www.yourdomain.com. On Cloudflare:

  1. Add the root domain (yourdomain.com) as a custom domain in Pages
  2. Add www.yourdomain.com as a second custom domain
  3. Cloudflare handles both and redirects www to root (or vice versa) automatically

Verifying It Works

After adding the custom domain, check the status:

# Check DNS resolution
$ dig yourdomain.com +short
# Should return a Cloudflare IP or your pages.dev CNAME

# Check HTTPS
$ curl -I https://yourdomain.com
# Look for HTTP/2 200 and cf-ray header

If the domain shows "Active" in the Pages dashboard but the site doesn't load yet, give it a few minutes for the SSL certificate to provision.

Common Issues

  • ERR_TOO_MANY_REDIRECTS — your SSL mode is set to "Flexible" instead of "Full". Go to SSL/TLS settings and set it to Full (strict)
  • DNS_PROBE_FINISHED_NXDOMAIN — DNS hasn't propagated yet, or the record is wrong. Double-check the CNAME target
  • Certificate not ready — can take up to 15 minutes. Cloudflare provisions a free certificate automatically
  • Old site still showing — clear your browser cache or test in incognito. DNS caching can also delay the switch
